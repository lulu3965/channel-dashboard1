# 产品渠道管理看板 - 使用指南

## 📊 如何修改数据

### 方式一：直接编辑 data.json（推荐）

1. 打开项目目录下的 `data.json` 文件
2. 按以下结构添加/修改你的产品和渠道数据：

```json
{
  "products": [
    {
      "id": "prod_001",
      "name": "你的产品名称",
      "image": null,
      "channels": ["短视频(达人)", "达播", "自播(单杯)", "商品卡(承接)", "货架", "商品卡(买一送一)", "破价达播", "自营短视频(达人号)", "自播(买一送一)挂链", "线下(小鹿王子)", "达播(买一送一)"],
      "data": {
        "ratio": ["5.60%", "2.80%", "42.02%", "5.60%", "5.60%", "1.40%", "22.41%", "2.80%", "0.56%", "8.40%", "2.80%"],
        "volume": ["20,000", "10,000", "150,000", "20,000", "20,000", "5,000", "80,000", "10,000", "2,000", "30,000", "10,000"],
        "sales": ["159.8万", "79.9万", "1198.5万", "159.8万", "159.8万", "50.0万", "399.2万", "79.9万", "20.0万", "105.0万", "79.9万"],
        "profit": ["-31.6万", "16.6万", "0.1万", "34.4万", "38.1万", "6.7万", "29.0万", "-10.2万", "-0.9万", "9.9万", "5.0万"]
      },
      "kpi": {
        "totalVolume": "320,000",
        "totalSales": "¥2,173.2万",
        "totalProfit": "¥184.1万",
        "profitRate": "整体利润率 8.5%",
        "bestChannel": "自播(单杯)",
        "bestChannelSub": "占比42.02% | 利润¥0.1万"
      }
    }
  ]
}
```

**字段说明：**
- `id`: 产品唯一标识（建议格式 prod_001, prod_002...）
- `name`: 产品名称
- `image`: 产品图片 Base64（可选，上传后自动填充）
- `channels`: 渠道名称数组（顺序对应下方数据）
- `data.ratio`: 各渠道单量占比（百分比字符串）
- `data.volume`: 各渠道单量
- `data.sales`: 各渠道销售额
- `data.profit`: 各渠道净利润
- `kpi`: 汇总指标（总单量、总销售额、总利润、利润率、最优渠道）

### 方式二：用 Excel 编辑后导出为 JSON

1. 在 Excel 中按以下列组织数据：
   - 第一行：渠道名称
   - 第二行：单量占比
   - 第三行：单量
   - 第四行：销售额
   - 第五行：净利润
2. 使用在线工具或脚本将 Excel 转为 JSON 格式
3. 替换 `data.json` 中的 `products` 数组内容

##  外网访问方案

当前看板运行在本地服务器（`localhost:8080`），仅限局域网访问。以下是三种外网部署方案，**推荐方案一**：

### 方案一：GitHub Pages（免费，最稳定）⭐

**优点：** 永久免费、无需服务器、全球 CDN 加速、自动 HTTPS  
**缺点：** 纯静态页面，无法保存用户填写的待办事项到服务器

**步骤：**
1. 注册 GitHub 账号（https://github.com）
2. 创建一个新仓库（如 `channel-dashboard`）
3. 将以下文件上传到仓库：
   - `product-channel-dashboard.html`
   - `data.json`
4. 进入仓库 **Settings → Pages**
5. Source 选择 **Deploy from a branch**，Branch 选 `main`，Directory 选 `/ (root)`
6. 点击 Save，等待 1-2 分钟
7. 获得公网地址：`https://你的用户名.github.io/channel-dashboard/`

**更新数据：** 直接在 GitHub 网页上编辑 `data.json`，或本地修改后 `git push`

---

### 方案二：Vercel / Netlify（免费，支持自定义域名）

**优点：** 自动部署、支持自定义域名、有简单的后端 API 能力  
**步骤：**
1. 注册 Vercel（https://vercel.com）或 Netlify（https://netlify.com）
2. 连接你的 GitHub 仓库
3. 自动部署，获得公网 URL（如 `https://channel-dashboard.vercel.app`）
4. 可在设置中绑定自己的域名

---

### 方案三：内网穿透（临时调试用）

**适用场景：** 快速分享给同事查看，不需要长期稳定  
**工具：** ngrok / cpolar / natapp

以 ngrok 为例：
```bash
# 安装 ngrok 后执行
ngrok http 8080
```
会生成一个临时公网地址（如 `https://xxxx.ngrok-free.app`），有效期约 2 小时。

## ⚠️ 重要提示

1. **待办事项数据保存在浏览器内存中**，刷新页面后会丢失。如需持久化，需要将数据写回 `data.json`（需后端支持）。
2. **图片上传后以 Base64 存储在内存中**，刷新也会丢失。如需保留，请将图片上传到图床（如 sm.ms、imgur），然后在 `data.json` 中将 `image` 字段改为图片 URL。
3. **修改 `data.json` 后刷新页面即可生效**，无需重启服务器。
4. 如果部署到 GitHub Pages，由于是纯静态站点，`fetch('data.json')` 可以正常工作；但如果有跨域问题，请确保两个文件在同一目录下。

## 🔧 常见问题

**Q: 修改 data.json 后页面没有变化？**  
A: 强制刷新浏览器（Ctrl+F5 / Cmd+Shift+R）清除缓存。

**Q: 页面显示"无法加载数据文件"？**  
A: 检查 `data.json` 是否存在于同一目录，且 JSON 格式正确（可用 https://jsonlint.com 验证）。

**Q: 如何添加更多产品？**  
A: 在 `data.json` 的 `products` 数组中添加新的产品对象，复制已有产品的结构并修改数据即可。

**Q: 外网访问时图片不显示？**  
A: Base64 图片在静态托管下可以正常显示。但如果图片太大（>1MB），建议上传到图床后用 URL 引用。
